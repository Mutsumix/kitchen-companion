# 2026-07-20(午後の部3): 縦一本貫通 — 音声往復の完成

## やったこと

1. Xtensaツールチェーンを1.95.0.0→1.97.0.0に更新(`espup install --toolchain-version 1.97.0.0`)→ ICE解消、`opt-level="z"` に復帰
2. それでも起動クラッシュ再発 → **真因はメインタスクのスタックオーバーフローと判明**(3コンパイラ設定で同一クラッシュ=自分のバグの証拠)。`CONFIG_ESP_MAIN_TASK_STACK_SIZE=20000` で解決
3. **タッチ録音→Worker(STT→LLM→TTS)→スピーカー再生の縦一本が実機で貫通**
4. UX改善: 3秒固定録音→**プッシュ・トゥ・トーク**(押している間録音・離して送信、上限15秒・0.5秒未満キャンセル)。白丸お絵かきギミック撤去。ビープ80msに短縮
5. 「不安定に再起動」の残バグ修正: I2S非ブロッキング書き込みのESP_ERR_TIMEOUTをエラー扱いしていた(タイムアウト=DMA満杯=正常)

## 計測値(実機・初回貫通時)

- 発話13.8秒(442KB WAV)→ **デバイス計測の往復8386ms**
  - Worker内訳: STT 1941ms / LLM 727ms / TTS 2115ms(計4783ms)
  - 残り約3.6秒 = TLSハンドシェイク+442KB上り+203KB下り(中継器Wi-Fi経由)
- 応答音声: 203KB PCM ≈ 6.4秒分
- 改善余地: 送信データの圧縮/ストリーミング、TTSストリーミング、TLSコネクション再利用
