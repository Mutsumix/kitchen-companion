# 2026-07-20: タッチ入力とWi-Fi接続

## やったこと

1. タッチ入力(FT6336)のポーリング実装 → タッチ位置に白丸描画で座標系検証
2. Wi-Fi接続(toml-cfgによる認証情報管理 + リトライ付き接続)→ 実機でIP取得

## タッチ入力(コミット `ede3e8b`)

新規クレート不要(既存のI2Cバスにいるアドレス0x38のFT6336を読むだけ)。

- レジスタ0x02から5バイト連続読み(タッチ点数、X上位/下位、Y上位/下位)
- 座標は12bit。上位バイトは下位4bitのみ有効
- 20ms間隔でポーリングし、タッチ位置に白丸を描画 → 指でなぞると軌跡が描け、座標系が画面と一致していることを目視確認

```sh
cargo build && cargo run -- --port /dev/cu.usbmodem1101 --non-interactive
```

## Wi-Fi接続(コミット `e167d65`)

### 認証情報の管理方式

リポジトリが公開のため、SSID/パスワードは `toml-cfg` クレートで gitignore 済みの `cfg.toml` から読む方式に(雛形 `cfg.toml.example` をコミット)。

```sh
cp cfg.toml.example cfg.toml
# cfg.toml にSSID/パスワードを記入(エディタで手動。チャット経由で秘密情報を扱わない)
# 注意: cfg.toml変更だけでは再コンパイルされないことがある
touch src/main.rs && cargo build
```

### 実装

- `BlockingWifi` + `EspWifi` で接続、最大5回リトライ(3秒間隔)
- 取得したIPアドレスをLCD画面下部に描画(シリアルに頼らない動作確認)

### 接続トラブルと解決

親機SSIDでは「認証OK→アソシエーションで切断→タイムアウト」を繰り返した。接続先を**中継器のSSID(`..._EXT`)に変更したら一発で接続**。デバイス設置場所から親機が遠く、中継器経由が正解だった。切り分けの詳細は [troubleshooting.md](../troubleshooting.md) 参照。

結果: 画面に `IP: 192.168.1.27` が表示され、接続成功。
