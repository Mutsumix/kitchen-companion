# docs の構成

このディレクトリは書籍(技術書典向け同人誌)の一次素材。以下のルールで記録する。

| ファイル/ディレクトリ | 役割 | 書くタイミング |
|---|---|---|
| [worklog/](worklog/) | 時系列の作業ログ。実際に打ったコマンド・出力の要点・計測値・失敗を含む順序をそのまま残す(本の「実況パート」の下書き) | マイルストーンごと(コミット前に更新) |
| [setup.md](setup.md) | 環境構築の再現手順。読者がコピペで追える清書版。失敗は含めず troubleshooting.md へのリンクで参照 | 環境が変わったとき |
| [troubleshooting.md](troubleshooting.md) | ハマりどころ。エラーメッセージは原文のまま残す(本の「ハマりどころコラム」の素材) | ハマったらすぐ |
| [decisions.md](decisions.md) | 設計判断と理由(本の「判断の言語化」の素材) | 判断したらすぐ |

## 運用ルール

- worklog は1作業日1ファイル: `worklog/YYYY-MM-DD-短いスラッグ.md`
- クラウド操作(Cloudflareダッシュボード等、コマンドにならないもの)は worklog に番号付き手順で記録し、スクリーンショットは `docs/images/` に置いて参照する
- レイテンシなどの計測値は必ず worklog に数値で残す(往復レイテンシの実測は書籍の重要データ)
- コミットメッセージと worklog が対応するように、マイルストーンのコミット前に worklog を更新する
