# 顔スプライトの入稿仕様

顔のビジュアルデザインは作者が担当し、デバイス側は受け取った画像を状態に応じて切り替える。

| 項目 | 仕様 |
|---|---|
| キャンバスサイズ | **320×155ピクセル**(顔エリア。画面320×240のうちy=30〜185) |
| ファイル形式 | **PNG**(透過なし。背景は黒 `#000000` ベタ塗り=画面の地色と一致) |
| 色 | フルカラー可(表示時にRGB565=約6.5万色へ変換。微細なグラデーションは段付きの可能性) |
| 基本セット | 4状態×1枚: `idle.png`(待機) / `listen.png`(聞き耳) / `think.png`(考え中) / `speak.png`(話し中) |
| アニメーション | 連番 `idle_0.png, idle_1.png, ...`(まばたき2〜3枚、口パク2〜3枚から) |
| 容量目安 | 1枚 320×155×2B ≈ 97KB(変換後)。フラッシュ空き約14MBのため数十枚可 |
| 置き場所 | `assets/face/` に置く(変換スクリプトとinclude_bytes!の組み込みはデバイス側担当) |

## 複数キャラクター対応(2026-07-20追加)

- 入稿は `assets/openai-output/faceN/` にキャラごとのディレクトリで置く
- 変換は `tools/face_convert.py` の `CHARACTERS` にエントリを追加(`align`: ベゼル基準の位置合わせをするか。枠構造がない絵柄はFalse=無補正)
- 変換結果は `assets/face/<キャラ名>/` に出力され、`src/main.rs` の `FACE_FRAMES` に `char_frames!("<キャラ名>")` を追加+`CHAR_COUNT`を更新
- 切替は画面左下の[顔]ボタン(選択はNVSに保存され再起動後も維持)
- 容量目安: 1キャラ12枚≒1.2MB(フラッシュ)。16MBに対して10キャラ程度まで現実的

## 実装メモ(デバイス側)

- 現在は `draw_face()` が状態名の文字(u8g2-fonts の日本語フォント)を描くプレースホルダ
- 画像化するときは PNG→RGB565 raw 変換スクリプトを `tools/` に追加し、`include_bytes!` でフラッシュに同梱、`embedded_graphics::image::ImageRaw` で描画する
- 口パクは再生中のRMS(すでに計測している)と連動させられる
